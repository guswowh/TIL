## - 익명, 기명 함수

> 기명함수
  - 이름이 있는 함수이다, 함수를 선언 할 수 있다.
> 익명함수
  - 이름이 없는 함수로 함수호출을 할 수 없다, 그래서 함수 표현으로만 사용 된다.

```js
function hello( ... ){ ... } // 기명함수

function ( ... ){ ... } // 익명함수
```

<br />

## - 콜백 함수(Callback function) === 함수 데이터
  - 함수의 매개변수를 함수로 호출 하여 반환값을 함수로 받는다
  ```js
  function 함수명(매개변수) {
  매개변수(1, 3) // 함수 호출
  }

  function 반환함수(a, b) {
    console.log(a + b) // 함수 표현
  }

  함수명(반환함수) // 함수 호출
  ```

## - 생성자 함수

  - 생성자 함수는 객체에 함수를 생성하여 함수의 중복을 막는다.
  - 생성자 함수는 영문 앞자리를 대문자로 표기한다 (ex: new User( ... ))
> 프로토타입 객체(prototype object)
  - 객체의 메소드와 속성을 상속 받기 위한 객체
> 인스턴스(instuns)
  - 생성자 함수 에서 반환된 객체 데이터
> 멤버(Member)
  - 객체의 프로퍼티 안에 속성과 메서드가 함께 있다는 표현
  <br /><br />
    > 멤버 패턴
    ```js
    const 멤버현재 = {
      성: '조',
      이름: '현재',
      성함: function () { // 축약형 : 성함() {
        return `${this.성}${this.이름}`
      }
    }
    console.log(멤버현재.성함())

    const 멤버현진 = {
      성: '조',
      이름: '현진',
      성함: function () {
        return `${this.성}${this.이름}`
      }
    }
    console.log(멤버현진.성함())

    const 멤버솔 = {
      성: '김',
      이름: '솔',
      성함: function () {
        return `${this.성}${this.이름}`
      }
    }
    console.log(멤버솔.성함())
    ```
    > 생성자 함수 패턴
    ```js
    function 멤버(성, 이름) {
    this.성 = 성
    this.이름 = 이름
    }

    멤버.prototype.성함 = function () {
      return `${this.성}${this.이름}`
    }
    
    const 인스턴스현재 = new 멤버('조', '현재')
    const 인스턴스현진 = new 멤버('조', '현진')
    const 인스턴스솔 = new 멤버('김', '솔')

    console.log(인스턴스현재.성함())
    console.log(인스턴스현진.성함())
    console.log(인스턴스솔.성함())
    ```

    > json 서버로 생성자 함수 호출 함수를 만드는 법! (자주 안씀.)

      ```js
      const sum = new Function('a', 'b', 'console.log(a + b)')
      sum(2, 4)
      ```

## - 함수 선언과 표현

<br />

> 구조 분해 할당(Destructuring assignment)

  - 구조 분해 할당 구문은 배열이나 객체의 속성을 분해하여 그 값을 변수에 담을 수 있게 하는 표현식

    문법 | 설명
    --|--
    const { 키 } = 객체명 | 기본 문법
    const { 키 = '기본값' } = 객체명 | 기본 값 문법
    const { 키: 기본값이름 = '기본값' } = 객체명 | 기본 값 이름 문법
    const { 키: 이름, 키: 이름 } = 객체명 | 객체 키 이름 바꾸는 문법

    > 변수 패턴
    ```js
    const user = {
      name: 'guswowh',
      age: '88',
      isValid: true
    }

    const { isValid } = user
    console.log(isValid)

    const { email = 'guswowh88@gmail.com' } = user
    console.log(email)

    // true
    // "guswowh88@gmail.com"

    const {
      name: e,
      age: a,
      isValid: i,
      email: x = 'the'
    } = user
    console.log(e, a, i)

    // "guswowh" "88" true
    ```

  - 함수 매개변수 패턴 vs 함수 구조분해 패턴
  
    > 함수 구조분해 할당 패턴
    ```js
    const 사용자 = {
      이름: '조현재',
      나이: 34
    }

    function 사용자이름({이름, 나이}){
      return [이름, 나이]
    }

    console.log(사용자이름(사용자))
    ```
    > 함수 매개변수 패턴
    ```js
    const 사용자 = {
      이름: '조현재',
      나이: 34
    }

    function 사용자이름(사용자){ // 변경
      const 이름 = 사용자.이름 // 추가
      const 나이 = 사용자.나이 // 추가
      return [이름, 나이]
    }

    console.log(사용자이름(사용자))
    ```

> 나머지 매개변수(Rest parameter)

## - this

- 일반 함수에서 `this`는 __호출되는 위치__ 에서 정의됨!
- 화살표 함수에서 `this`는 자신이 선언된 __함수(렉시컬) 범위__ 에서 정의됨
- 렉시컬영역 : 자신을 감싸고 있는영역

> 일반함수 범위 패턴
```js
const 사용자 = {
  나이: 85,
  내나이: function () {
      console.log(this.나이)
    }
}

사용자.내나이()
```

> 화살표함수 범위 패턴

```js
const 사용자 = {
  나이: 85,
  내나이: function () {
    setTimeout(() => {
      console.log(this.나이)
    }, 1000)
  }
}

사용자.내나이()
```

## - ES6 Classes : 생성자 함수 패턴보다 짧게 구문을 완성 할 수 있다.

  > 생성자 함수 패턴
  ```js
  function 멤버(성, 이름) {
  this.성 = 성
  this.이름 = 이름
  }

  멤버.prototype.성함 = function () {
    return `${this.성}${this.이름}`
  }
  
  const 인스턴스현재 = new 멤버('조', '현재')
  const 인스턴스현진 = new 멤버('조', '현진')
  const 인스턴스솔 = new 멤버('김', '솔')

  console.log(인스턴스현재.성함())
  console.log(인스턴스현진.성함())
  console.log(인스턴스솔.성함())
  ```
  > constructor : 클레스의 인스턴스 객체를 생성하고 초기화 하는 메서트
  ```js
  class 멤버 {
    constructor(성, 이름) { // constructor function (){} 의 축약형
      this.성 = 성
      this.이름 = 이름
    }
    성함() {
      return `${this.성}${this.이름}`
    }
  }

  const 인스턴스현재 = new 멤버('조', '현재')
  const 인스턴스현진 = new 멤버('조', '현진')
  const 인스턴스솔 = new 멤버('김', '솔')

  console.log(인스턴스현재.성함())
  console.log(인스턴스현진.성함())
  console.log(인스턴스솔.성함())
  ```

## - ES6 Classes : 상속 / 확장
  > extends : 부모 클래스의 자식으로 만들기 위해 class 선언 또는 class 식에 사용됨
  - 문법 : `class 부모클레스 extends 자식클레스 { ... }`
  > super : 부모 객체의 함수를 호출할 때 사용됩니다.
  - 문법 : `super( 인수 )`

  > 상속 / 확장 패턴
  ```js
  class 자동차 {
    constructor(이름, 바퀴) {
      this.이름 = 이름
      this.바퀴 = 바퀴
    }
  }

  const 내자동차 = new 자동차('운송수단', 2)
  console.log(내자동차)

  class 자전거 extends 자동차 {
    constructor(이름, 바퀴) {
      super(이름, 바퀴)
    }
  }

  const 내자전거 = new 자전거('삼천리', 2)
  const 동생자전거 = new 자전거('새발', 3)
  console.log(내자전거)
  console.log(동생자전거)

  class 차 extends 자동차 {
    constructor(이름, 바퀴, 면허) {
      super(이름, 바퀴)
      this.면허 = 면허
    }
  }

  const 내차 = new 차('벤츠', 4, true)
  const 부모차 = new 차('포르쉐', 4, false)
  console.log(내차)
  console.log(부모차)
  ```
